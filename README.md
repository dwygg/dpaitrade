# dpaitrade

`dpaitrade` 是一个面向研究和回测的 AI 交易工程骨架。

当前版本的核心目标不是做“AI 自由发挥下单”，而是先搭建一套可验证、可回测、可迭代的交易系统基础设施：

- 规则模块负责识别市场状态与生成候选信号
- AI 模块负责过滤、评分、路由与解释
- 风控模块负责最终审批
- 回测模块负责成交仿真与绩效评估

---

## 一、设计原则

本项目遵循以下原则：

1. **先有规则基线，再引入 AI**
   - 先保证纯规则系统可运行、可回测
   - 再引入 AI 做“增量价值”测试

2. **AI 不直接掌控风控**
   - 单笔风险
   - 最大回撤
   - 最大持仓
   - 点差过滤
   - 冷却机制

   上述规则必须是确定性的，不交给 AI 决定。

3. **AI 的职责必须可测**
   - 它是分类器？
   - 过滤器？
   - 策略路由器？
   - 风险建议器？

   每种职责都必须能单独评估。

4. **日志、注释、描述统一使用中文**
   - 便于后续研发和排查

## 二、v0 阶段目标

第一阶段只做下面这条链路：

```text
D1/H4/M15 规则模块
↓
生成 CandidateSignal（候选信号）
↓
Agent 模块给出 allow_trade / setup_score / reason
↓
Risk 模块审批
↓
Execution / Backtest 模块记录结果
↓
Evaluation 模块比较：
- 纯规则基线
- 规则 + AI 过滤
- 规则 + 简单阈值过滤
